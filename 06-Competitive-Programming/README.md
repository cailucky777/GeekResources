# Competitive Programming Module

## Why Competitive Programming Matters

Competitive programming is one of the most effective ways for students to sharpen their problem-solving skills, deepen their understanding of algorithms and data structures, and build confidence under pressure. Benefits include:

- **Stronger algorithmic thinking** -- Students learn to break down problems into solvable components.
- **Faster coding fluency** -- Timed contests push students to write correct code quickly.
- **College and career readiness** -- Contest experience stands out on university applications and is valued in technical interviews.
- **Community and motivation** -- Competing with peers locally and internationally gives students purpose and excitement.
- **Mathematical reasoning** -- Many problems require creative applications of math, strengthening interdisciplinary skills.

## Getting Started

### Prerequisites

- Comfortable with at least one programming language (we use **Swift** in this curriculum, but C++ and Python are also common in contests).
- Understanding of basic data types, loops, conditionals, functions, and arrays.
- Familiarity with reading input from standard input and writing output to standard output.

### Setting Up Your Environment

1. **Swift**: Use Terminal or Xcode. Compile and run with `swift filename.swift`.
2. **C++**: Use any text editor plus `g++` for compilation, or an online judge IDE.
3. **Online judges**: Create accounts on [DMOJ](https://dmoj.ca/), [Codeforces](https://codeforces.com/), and [LeetCode](https://leetcode.com/).

## Recommended Progression

### Phase 1: Foundations (Weeks 1-4)

**Difficulty: Easy**

Focus on building confidence with straightforward problems.

- Basic I/O and string manipulation
- Simple math (divisibility, modular arithmetic, GCD)
- Linear search and basic sorting
- Simple loops and conditionals

**Practice**: See [Beginner Problems](problems/Beginner-Problems.md)

### Phase 2: Core Algorithms (Weeks 5-10)

**Difficulty: Medium**

Introduce fundamental algorithms and data structures.

- Binary search
- Two pointers technique
- Stacks, queues, and linked lists
- Recursion and backtracking
- Basic dynamic programming (Fibonacci, coin change)
- Greedy algorithms
- Simple graph traversal (BFS, DFS)

**Practice**: CCC Junior problems, USACO Bronze problems, LeetCode Easy/Medium.

### Phase 3: Advanced Techniques (Weeks 11-16)

**Difficulty: Hard**

Tackle challenging problems requiring deeper insight.

- Advanced dynamic programming (knapsack, interval DP)
- Graph algorithms (Dijkstra, MST, topological sort)
- Segment trees and binary indexed trees
- String algorithms (KMP, hashing)
- Number theory (primes, Euler's totient)
- Computational geometry basics

**Practice**: CCC Senior problems, USACO Silver/Gold problems, Codeforces Div 2.

## Tips for Teachers

1. **Start with fun problems** -- Choose problems with engaging stories or relatable contexts.
2. **Pair programming** -- Let students work in pairs initially to build confidence.
3. **Weekly problem sets** -- Assign 3-5 problems per week at the appropriate difficulty level.
4. **Mock contests** -- Run timed practice sessions in class to simulate real contest conditions.
5. **Celebrate progress** -- Track rating improvements and celebrate when students solve harder problems.
6. **Form a team** -- Register for CCC and ECOO as a school -- the team experience is valuable.
