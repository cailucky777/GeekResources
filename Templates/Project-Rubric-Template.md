## Rubric

| Category | Exceeds Expectations (4) | Meets Expectations (3) | Approaching Expectations (2) | Beginning (1) | Score |
|----------|--------------------------|------------------------|------------------------------|---------------|-------|
| **Functionality** | All required features work correctly with no bugs. Includes additional features beyond requirements. Handles edge cases gracefully. | All required features work correctly with minor bugs that do not affect usability. Most edge cases handled. | Most required features work but some have noticeable bugs. Several edge cases are not handled. App may crash in certain scenarios. | Few required features work. Major bugs or crashes prevent normal use. Core functionality is incomplete. | /4 |
| **Code Quality** | Code is clean, well-organized, and follows Swift conventions. Meaningful variable/function names throughout. Proper use of functions, structs, and protocols to avoid repetition. Includes helpful comments. | Code is mostly well-organized and follows conventions. Good variable/function names. Some use of functions and structs to organize code. Adequate comments. | Code works but is disorganized in places. Some unclear naming. Limited use of functions -- some code is repeated. Few or no comments. | Code is disorganized and hard to follow. Poor naming conventions. Significant code duplication. No comments. Does not follow Swift conventions. | /4 |
| **UI/UX Design** | Interface is polished, intuitive, and visually appealing. Layout adapts well to different screen sizes. Consistent styling and spacing. Thoughtful use of color, typography, and icons. | Interface is clean and functional. Layout works well on target device. Mostly consistent styling. Good usability. | Interface is functional but rough. Some layout issues or inconsistent styling. Usable but not intuitive in places. | Interface is incomplete or very rough. Major layout problems. Poor usability -- user would struggle to navigate the app. | /4 |
| **Creativity & Problem-Solving** | Demonstrates original thinking and creative approaches. Goes well beyond the minimum requirements. Shows strong problem-solving skills in implementation choices. | Shows some creativity in design or implementation. Meets all requirements with a personal touch. Good problem-solving demonstrated. | Meets minimum requirements with little personalization. Limited evidence of creative thinking. Basic problem-solving approach. | Does not meet minimum requirements. Little effort to personalize or think creatively. Relied heavily on copying examples without adaptation. | /4 |
| **Documentation & Presentation** | Thorough README with clear description, setup instructions, screenshots, and reflection. Excellent presentation -- clearly explains design decisions and what was learned. | Good README with description and basic instructions. Solid presentation covering main features and some reflection. | Minimal README with incomplete information. Presentation covers basics but lacks depth or reflection. | No README or very incomplete. Presentation is missing, minimal, or does not demonstrate understanding of the project. | /4 |
