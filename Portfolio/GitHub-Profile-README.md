# Your GitHub Profile README

The single highest-leverage 30 minutes in your portfolio. It's the first thing anyone sees on your profile.

---

## How It Works

GitHub looks for a **special repo** with the same name as your username, containing a `README.md`. Its contents appear on your profile page.

1. Create a new repo named **exactly your username** (e.g., `alice-chen/alice-chen`). GitHub shows a ✨ note "This is a special repository."
2. Add a `README.md`.
3. Commit.

That's it — your profile now has a custom intro.

---

## A Template That Works

```markdown
# Hi, I'm Alice 👋

Sophomore at Lincoln High, building iOS apps and learning algorithms.

### What I'm working on

- 📱 **Flashcards Pro** — a spaced-repetition app in SwiftUI
- 🎮 **Pixel Dungeon** — a 2D rogue-like in Unity
- 🧮 **LeetCode streak** — 150+ problems and counting

### Tools I use

Swift · SwiftUI · Unity · Python · Git

### Connect

- 🌐 [alice-chen.dev](https://alice-chen.dev)
- ✉️ alice@example.com
```

Keep it short. Nobody reads long profile READMEs.

---

## What to Include

1. **One sentence about you.** School, year, what you love building.
2. **2–3 current projects** with links.
3. **Tools you actually use** (don't pad — listing every language you've touched once looks insecure).
4. **One way to reach you.**

---

## What to Leave Out

- "I'm passionate about coding!" — everyone says this. Show, don't tell.
- Streaks and "GitHub stats" cards — they look cool for 5 seconds, then feel generic.
- A long personal story — save it for your website.
- "Looking for internships" on your high-school profile. It reads as desperate. Just make cool things.

---

## Pin Your Best Repos

Below the README, GitHub lets you **pin 6 repositories**. This is your portfolio inside your portfolio.

Rules:

- Pin only repos with a good README and screenshot.
- Don't pin forks unless your contribution is obvious.
- Diverse is better than uniform — a Swift app, a Python tool, a game, and a writeup is more interesting than 6 clones of "Calculator."
